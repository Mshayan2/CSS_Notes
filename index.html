What is CSS & History

Cascading Style Sheets (CSS) is the design sheet language used to control the visual presentation of web documents written in HTML. It coordinates color grids, layouts, typography, shapes, and response animations.

The Cascading Mechanism

When multiple CSS rules conflict, the browser resolves them using three primary factors:

  • Specificity: Calculation of selector weight (ID > Class > Element).
  • Inheritance: Properties that automatically pass down from parents to children (e.g. color, font-family).
  • Order of Appearance: Later rules override earlier ones if specificity is equal.
  • Importance (!important): Overrides all other declarations (use sparingly).

Widget B: Ways to Add CSS

Applied directly within the HTML element using the style attribute. Highly discouraged for large scale projects as it duplicates markup and bypasses caching benefits.

<p style="color: #ff0055; font-size: 18px;">Hello World</p>

Placed within a <style> block inside the HTML <head>. Ideal for single-page files or isolated styling requirements.

<head>
  <style>
    p {
      color: cyan;
      line-height: 1.6;
    }
  </style>
</head>

Written in a separate .css stylesheet and linked via the <link> tag. Best practice for maintainable, multi-page websites as it allows resource caching.

<head>
  <link rel="stylesheet" href="styles.css">
</head>

CSS Selectors & Combinators

Selectors designate which HTML elements receive specific rules. Mastering them prevents styling collision.

Basic Selectors

  • Element Selector: Targets tags directly (e.g. p, h1). Applies globally.
  • Class Selector: Targets matching class attributes (prefixed with ., e.g. .card). Reusable on multiple elements.
  • ID Selector: Targets a single element (prefixed with #, e.g. #main-header). Must be unique to a page.
  • Universal Selector (*): Selects all elements. Commonly used for css resets to zero out margins and padding.

Combinators & Grouping

  • Grouping Selector: Combines selectors separated by commas (e.g. h1, h2, p) to share a common code declaration block.
  • Descendant Selector (space): Selects children nested anywhere inside a parent element (e.g. .card p).
  • Child Selector (>): Selects elements that are direct children of the parent (e.g. .card > h2).
  • Adjacent Sibling Selector (+): Targets the element immediately following another sibling element (e.g. h1 + p).
/* Selector Combinator Example */
.card > p {
  color: #a855f7;
  margin-bottom: 12px;
}

Colors, Typography & Units

Color Definition Formats

CSS supports several syntax patterns to declare colors:

  • Hexadecimal: 6 characters prefixed with # representing Red, Green, Blue levels (e.g. #00e5ff). You can append 2 characters for opacity (e.g. #00e5ff80 for 50% opacity).
  • RGB & RGBA: Red, Green, Blue levels from 0 to 255 (e.g. rgb(0, 229, 255)). RGBA adds a fourth transparency alpha value between 0.0 and 1.0 (e.g. rgba(0, 229, 255, 0.5)).
  • HSL & HSLA: Hue (0-360 degrees), Saturation (0-100%), and Lightness (0-100%). Great for cohesive theme creation.

CSS Measurement Units

  • px (Pixels): Absolute physical screen dots. Fixed size, does not scale with browser font changes.
  • em: Relative to the font size of its parent element. Compounds when nested.
  • rem (Root em): Relative to the root <html> element's font size. Safer and highly recommended for layout consistency and accessibility.
  • % (Percentage): Relative to parent size parameters.
  • vw / vh (Viewport Width/Height): Equal to 1% of the viewport width or height. Useful for full-screen banners.

Backgrounds & Borders

Background Properties

  • background-color: Fills element background (e.g. solid color, linear-gradients, or radial-gradients).
  • background-image: Loads image URL assets inside the container viewport.
  • background-repeat: Handles wrapping tiling behaviors (no-repeat, repeat-x, repeat-y).
  • background-size: Adjusts image sizing behavior:
    • cover: scales to cover element (some cropping may occur).
    • contain: scales to fit element without cropping.
  • background-position: Positions image (e.g. center, top right).
.hero-banner {
  background-image: url('bg.jpg');
  background-size: cover;
  background-position: center;
}

Borders & Corners

  • border: Shorthand syntax combining width, style, and color: border: 2px solid #fff;.
  • border-style: Border patterns (solid, dashed, dotted, double).
  • border-radius: Rounds corners (e.g., 8px for buttons, 50% for circular avatars). Takes up to 4 values for individual corners clockwise.

Display & Positioning Styles

Display Properties

Determines how an element behaves in the document layout flow:

  • block: Spans full parent width. Starts on a new line (e.g. <div>, <p>, headings).
  • inline: Fits content size only. Cannot have width/height or top/bottom margins (e.g. <span>, <a>, <strong>).
  • inline-block: Flows like inline but allows setting custom width/height and padding.
  • none: Removes the element entirely from the document layout flow. Contrast with visibility: hidden which hides the element but keeps its layout space.

Positioning Methods

  • static: Default flow. Follows natural HTML arrangement.
  • relative: Offset from its normal position without altering surrounding elements. Acts as reference for absolute positioned children.
  • absolute: Positioned relative to the nearest positioned ancestor (relative, absolute, or fixed). Removed from normal flow.
  • fixed: Anchored relative to the browser viewport window. Stays fixed during scroll.
  • sticky: Hybrid. Acts relative until a threshold (e.g. top: 0) is reached during scrolling, then locks.

CSS Grid Layout

CSS Grid is a powerful 2-dimensional layout engine. While Flexbox is ideal for aligning items along a single axis (row OR column), Grid is perfect for arranging columns and rows simultaneously.

Key Properties

  • grid-template-columns: Defines number and size of columns (e.g. 1fr 1fr 1fr for three equal columns).
  • grid-template-rows: Defines heights of grid rows.
  • gap: Gutters spacing between rows and columns.
  • grid-column / grid-row: Tells child items how to span across columns or rows (e.g. 1 / 3).
  • repeat() & minmax(): Create responsive grids dynamically: grid-template-columns: repeat(auto-fit, minmax(200px, 1fr));.

Pseudo-classes & Pseudo-elements

Pseudo-classes (Single Colon :)

Used to style elements in specific states, like cursor hovers or when form inputs are selected:

  • :hover - Matches when the user hovers over the element.
  • :focus - Matches when an element receives focus (e.g. clicking text fields).
  • :active - Active at the exact moment of mouse click.
  • :nth-child(n) - Matches specific child index patterns (e.g. :nth-child(odd), :nth-child(3)).

Pseudo-elements (Double Colon ::)

Used to style specific parts of an element, or inject decorative content without HTML code changes:

  • ::before - Inserts generated content before the actual content of the selector. Must include the content: "" property.
  • ::after - Inserts content after the actual content of the selector.
  • ::placeholder - Styles the text inside empty inputs.
h1::before {
  content: "🔥 ";
  color: orange;
}

Transitions & Transforms

CSS Transitions

Smoothly changes style values over a given duration, preventing instant jarring snaps.

  • transition-property: Target values to watch (e.g., background-color, transform, or all).
  • transition-duration: Timing span (e.g. 0.3s, 500ms).
  • transition-timing-function: Velocity curve (ease-in-out, linear, or custom cubic-bezier).
  • transition-delay: Wait time before starting animation.

Responsive Design & Media Queries

Responsive web design ensures that websites display correctly on all screen resolutions, viewport widths, and devices (mobile, tablet, desktop).

Common Responsive Breakpoints

  • Mobile Devices: max-width: 480px
  • Tablets & Phablets: max-width: 768px
  • Laptops / Small Desktops: min-width: 1024px

Mobile-First Design: Writing default styles for small screens first, then wrapping larger desktop overrides inside @media (min-width: 768px). Recommended for performance.

Media Query Syntax

Styles inside this bracket apply only when conditions are met:

/* Mobile-First Adaptations */
@media only screen and (max-width: 768px) {
  .grid-container {
    grid-template-columns: 1fr; /* stack items */
  }
}
